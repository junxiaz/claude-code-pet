# 待确认问题清单(Open Questions)

配套《需求分析报告.md》。每条含:背景 / 为什么重要 / 不同答案对方案的影响 / **建议默认值**。按优先级分组。

---

## P0 —— 阻塞 v0.2 范围冻结,必须先答

### Q1. 是否收敛"自动放行 Read"的默认行为?(对应 FR-06 / 风险 R1)
- **背景**:`cc-autoallow.js` 目前默认自动放行 `Read/Glob/Grep`,写入用户级 `settings.json`,对本机所有 Claude 会话生效。`Read` 可无确认读取 `~/.ssh`、`.env`、凭证等任意文件并进入模型上下文。
- **为什么重要**:这是全项目最高严重度的安全隐患,直接影响谨慎用户的采用意愿与维护者的安全责任。
- **不同答案的影响**:
  - 维持现状 → 体验最省心,但安全叙事弱,面向社区分发有声誉风险。
  - 仅放行 Glob/Grep,Read 恢复确认 → 安全显著提升,略增确认次数。
  - Read 放行但排除敏感路径清单 → 折中,实现复杂度中等。
- **建议默认值**:**默认只自动放行 Glob/Grep;Read 需用户在 `cc-pet.json` 显式开启(opt-in),开启项附敏感路径排除**。安装时明确告知。

### Q2. 项目定位是"个人自用工具"还是"面向社区分发"?
- **背景**:仅 3 次提交、已有中英文 README + 徽章 + demo,呈现出对外分发意图,但无测试/CI/版本。
- **为什么重要**:定位决定 T6(测试/CI/版本)、T9(安全文档)的优先级与投入下限。
- **不同答案的影响**:自用 → 可弱化 CI/文档,聚焦体验;社区分发 → 安全默认(Q1)、可还原性、版本兼容成为硬要求。
- **建议默认值**:**按"面向社区分发"要求**(已具备分发外观),将 T6/T9 纳入 v0.2 Must。

## P1 —— 影响 v0.2/v0.3 设计,尽快答

### Q3. 依赖 `claude agents --json` 的稳定性预期如何?(对应 G4 / T4 / R3)
- **背景**:核心会话清单完全依赖该 CLI 命令,无版本/字段校验,失败时静默。
- **为什么重要**:上游一改格式即整体失效,且用户不可知。
- **不同答案的影响**:视为稳定 → 只加降级提示;视为易变 → 需字段校验 + 兼容层。
- **建议默认值**:**加字段校验 + 缺失时挂件显示 ⚠ 与"请升级 Claude Code"提示**(轻量,不做本地重实现)。

### Q4. 需要支持哪些终端 / 是否需要跨平台?(对应 G3 / T3 / CON-01)
- **背景**:当前仅 macOS + iTerm2/Terminal.app;Warp/WezTerm/kitty/VS Code 终端/tmux 全不支持。
- **为什么重要**:决定 T3 投入与潜在用户覆盖面。
- **不同答案的影响**:只保 2 终端 → 维护最轻;扩到 5+ → 需 adapter 抽象层,覆盖翻倍。
- **建议默认值**:**v0.3 先抽 adapter 接口并加 Warp + VS Code 终端**;跨平台暂不做(受 AppleScript/Hammerspoon 约束)。

### Q5. 空闲能耗与实时性如何取舍?(对应 G7/G8 / NFR-03 / T7)
- **背景**:动画常驻 14fps、轮询固定 2s、每 2s 全量遍历终端窗口取标题。
- **为什么重要**:笔记本用户在意耗电;窗口多时 osascript 可能卡顿。
- **不同答案的影响**:保持现状 → 实现简单、体验实时;引入节流 → 省电但代码更复杂。
- **建议默认值**:**动态态 14fps、纯 idle 降到 ≤1fps;轮询间隔与外观常量迁入 `cc-pet.json` 可配**。

## P2 —— 体验细节,可延后

### Q6. 超过 8 个会话时如何处理?(对应 G10 / FR-08)
- **背景**:`MAX_CHIPS=8`,第 9+ 会话被静默截断。
- **为什么重要**:重度用户可能同时开很多会话,信息丢失且无提示。
- **不同答案的影响**:保持截断 → 简单;显示 "+N" 或分页 → 信息不丢。
- **建议默认值**:**显示 "+N" 溢出提示**,MAX_CHIPS 可配。

### Q7. 多显示器场景的挂件定位策略?(对应 G11 / NFR-06)
- **背景**:仅用 `primaryScreen`,anchor 持久化后换屏可能落屏外。
- **为什么重要**:多屏用户可能"找不到挂件"。
- **不同答案的影响**:不处理 → 偶发丢失;加回弹校验 → 稳健。
- **建议默认值**:**anchor 越界时自动回弹到当前主屏可见区**。

### Q8. Notification 分类是否需要多语言?(对应 G5 / T5 / R4)
- **背景**:needs/waiting 判定依赖英文关键词正则,非英文通知会误判为 waiting(丢红色告警)。
- **为什么重要**:中文/其他语言用户可能漏掉"等你审批"。
- **不同答案的影响**:仅英文 → 覆盖不全;多语言或改用结构化字段 → 更准。
- **建议默认值**:**优先改用 hook 结构化字段判定,关键词作兜底并补中文**。

### Q9. 安装备份文件是否需要轮转清理?(对应 G9 / R6)
- **背景**:每次 `install.sh` 生成 `settings.json.bak.<ts>`,反复安装堆积。
- **为什么重要**:长期占磁盘(已 gitignore,不影响仓库)。
- **不同答案的影响**:不清理 → 无害但堆积;保留最近 N 份 → 整洁。
- **建议默认值**:**仅保留最近 3 份备份,其余自动清理**。
